### 發現的 Bug（3 個）
1. **`chatAccumulatedFiles` 未宣告**（handler.js）— `new Map()` 從未建立，try-catch 吃掉 ReferenceError
2. **`filled_form` 事件被忽略**（poller.js）— `fetchNewEvents` 只處理 `message` 和 `file`，prechat form 的 `filled_form` 事件完全被跳過
3. **Prechat 資料沒注入 AI 上下文** — `getChat()` 取的 `customer` 物件不含 prechat 欄位；欄位在 `filled_form` 事件的 `fields` 陣列裡

### 修改的檔案（VPS /opt/cs-bot/）
- **handler.js**：新增 `chatPrechatData` Map，從 `filled_form` 事件解析 Username + Inquiry Type，注入 AI system message，個人化歡迎訊息
- **poller.js**：`fetchNewEvents` 新增 `filled_form` 事件解析，呼叫 handler 的 `setPrechatData`
- **ai.js**：修復 SYSTEM_PROMPT 裡 markdown ``` 導致的模板字串語法錯誤（backtick 衝突）
- **knowledge/brand-config.md**：語言規則改為「ALWAYS respond in ENGLISH」（與 ai.js prompt 同步）

### Docker 踩坑
- `src/` 目錄是 build 進 image 的，不是 volume mount → 改了本機檔案但 container 用舊版
- 解法：docker-compose.yml 加 `./src:/app/src` volume mount
- `docker compose build` 記憶體不夠被 kill → 改用 volume mount 繞過
- 備份在 `/opt/cs-bot/src/*.bak`
